📋Pré-Requisitos
▼
🔴 Obrigatório
  • CPF ou CNPJ do cliente (válido)
  • Nome completo do cliente
  • Unidade / espaço associado ao acesso
  • Contrato ativo ou confirmação de vínculo
  • Data de início do acesso
  • Foto do cliente — enviada junto com o formulário (Control ID) ou não necessária (UniFi)
  • E-mail do cliente (obrigatório no UniFi)
🟡 Importante
  • Telefone de contato
  • Data de fim do acesso (se aplicável)
⚠️ Todos os dados chegam preenchidos pelo próprio cliente via formulário. O CPF é obrigatório para o cadastro no condomínio.
🟦Passo a Passo — Jestor
▼

Sistema de gestão interna. Ponto de entrada obrigatório para qualquer cadastro.

1
Acessar o Jestor e ir para o Pipeline de Cadastro de Acesso
Como fazer
Acesse app.jestor.com → entre com suas credenciais → localize OPS | Pipeline de Cadastro de Acesso
🗂️ Pipeline — Cadastro de Acesso abrir →
ℹ️ O pipeline e os formulários já estão configurados. Não crie novos campos ou estruturas sem alinhamento com a equipe.
2
Localizar o card no pipeline
O que fazer
O cliente preenche o formulário e o card já chega criado automaticamente com status Nova Solicitação. A colaboradora não preenche o formulário — apenas localiza e trata o card.
SE
dados completos e corretos
→
ENTÃO
Responsável da unidade move para Cadastro Autorizado
SE
dados incompletos ou incorretos
→
ENTÃO
Não avançar — acionar o responsável antes de qualquer ação
⚠️ Só avance para o sistema de acesso após o card estar em Cadastro Autorizado.
3
Identificar o sistema da unidade
O que fazer
Consultar a tabela de perfil operacional para saber qual sistema a unidade usa e se há recepcionista
📋 Tabela — Perfil Operacional abrir →
Com base no perfil operacional, siga para:
SE
sistema = UniFi
→
ENTÃO
Acionar Ops → seção UniFi
SE
sistema = Control ID + tem recepcionista
→
ENTÃO
Atendimento segue → seção Control ID
SE
sistema = Control ID + sem recepcionista
→
ENTÃO
Acionar Ops → seção Control ID
4
Selecionar o formulário correto
Formulários disponíveis
🏢 Formulário — Empresa XP abrir → 🏢 Formulário — Unidades de Coworking abrir → 🏢 Formulário — Prudential abrir →
⚠️ Se o cliente não se encaixa em nenhum dos 3 formulários, não avance — acione a equipe antes de cadastrar.
5
Acompanhar o pipeline
Etapas do pipeline
🔵 Nova Solicitação
Automação. Card criado ao enviar o formulário. Não mover manualmente.
✅ Cadastro Autorizado
Movido pelo responsável da unidade após validar os dados. Pronto para o sistema de acesso.
⏳ Control ID / UniFi Pendente
Movido ao iniciar o cadastro no sistema de acesso. Processo em andamento.
📷 Foto não aprovada
Automação: e-mail disparado ao cliente. Quem aprova: o próprio Control ID. Aplicável somente a unidades Control ID.
✅ Aprovada → Control ID / UniFi Concluído
❌ Rejeitada → manter aqui — automação reenvia e-mail
✅ Control ID / UniFi Concluído
Movido após cadastro concluído no sistema de acesso. Se não há etapa de condomínio, avançar para Finalizado.
🏛️ Condomínio Pendente
Canal varia por unidade. Consulte a tabela de perfil operacional.
📧 E-mail → automação dispara ao mover o card.
📱 App ou Portal → não é responsabilidade do time de atendimento.
✅ Condomínio Concluído
Atualizado automaticamente após envio do e-mail. Avançar para Finalizado.
🏁 Finalizado
Processo completo. Acesso liberado e tudo registrado.
🚫 Negado
Decisão do responsável da unidade. Regras de liberação devem ser alinhadas com a liderança previamente.
👤 Pessoa Inativa
Acionado quando a empresa informa desligamento.
  1. Mover card para Pessoa Inativa
  2. Consultar e-mail do condomínio na tabela de perfil operacional → coluna E-mail
  3. Enviar e-mail ao condomínio informando o bloqueio
  4. Inativar no sistema de acesso da unidade (Control ID ou UniFi)
⚠️ Não inative sem confirmação da empresa.
🔁 Duplicado
Cliente já existe no sistema de acesso. Não criar nada — encerrar o processo aqui.
ℹ️ Etapas com automação não precisam de ação manual além de mover o card para o status correto.
🟣Passo a Passo — UniFi
▼

Sistema de controle de acesso para unidades UniFi. Executado exclusivamente por Ops (Ludmilla ou Luana).

🟣 O UniFi não exige busca prévia — o sistema avisa pelo e-mail se a pessoa já está cadastrada. O e-mail é obrigatório.
1
Acessar o UniFi
O que fazer
Entrar no portal UniFi com suas credenciais
🔐 Login — UniFi abrir →
⚠️ Acesso restrito a Ops. Somente Ludmilla e Luana têm credenciais de acesso.
2
Selecionar a unidade correta
O que fazer
No menu superior esquerdo, selecionar a unidade correspondente ao cadastro
Como fazer
Clique no seletor de site (ex: XP - BELÉM / PA - UDM SE) → selecione a unidade correta
⚠️ Confirme que está na unidade correta antes de criar o usuário. Cadastro na unidade errada libera acesso no lugar errado.
3
Criar novo usuário
Como fazer
No painel lateral → Usuários ativos → clique no + azul → abre o formulário Criar novo usuário
Campos a preencher
Primeiro nomePrimeiro nome — conforme Jestor
SobrenomeRestante do nome — conforme Jestor
E-mailObrigatório. O sistema avisa se o e-mail já está cadastrado — não criar duplicata.
ℹ️ Se o UniFi alertar que o e-mail já existe, volte ao Jestor e mova o card para Duplicado.
4
Vincular ao grupo da unidade
O que fazer
Na seção Grupos do formulário → clicar em Selecionar grupos → selecionar o User Group padrão da unidade
Como saber o grupo correto
Consulte a tabela de perfil operacional — campo User Group padrão da unidade
⚠️ Não pule essa etapa. Sem o grupo vinculado, o cliente não terá as políticas de acesso da unidade aplicadas.
5
Criar e confirmar
Como fazer
Revisar os dados → clicar em Criar → confirmar que o usuário aparece na lista de Usuários ativos
ℹ️ Não é necessário adicionar credenciais (Touch Pass, NFC, PIN) neste momento — o processo padrão usa apenas o grupo de acesso.
6
Atualizar o pipeline no Jestor
Como fazer
Voltar ao Jestor → mover o card para Control ID / UniFi Concluído
ℹ️ Só mova o card após confirmar que o usuário foi criado e está visível na lista do UniFi.

Buscar se o cliente já existe

ℹ️ Mantenha o filtro em Ativos.
SE
pessoa encontrada
→
ENTÃO

SE
pessoa não encontrada
→
ENTÃO
Clicar em Nova Pessoa (botão azul, canto superior direito) → Passo 3
⚠️ Pesquise por nome e sobrenome — só o nome pode gerar falso negativo.
