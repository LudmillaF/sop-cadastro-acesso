📋Pré-Requisitos
▼
🔴 Obrigatório
  • CPF ou CNPJ do cliente (válido)
  • Nome completo do cliente
  • Unidade / espaço associado ao acesso
  • Contrato ativo ou confirmação de vínculo
  • Data de início do acesso
  • Foto do cliente (enviada junto com o formulário)
🟡 Importante
  • E-mail do cliente
  • Telefone de contato
  • Data de fim do acesso (se aplicável)
⚠️ Todos os dados chegam preenchidos pelo próprio cliente via formulário. O CPF é obrigatório também para o cadastro no condomínio.
🟦Passo a Passo — Jestor
▼

Sistema de gestão interna. Aqui fica o registro central do cliente.

1
Acessar o Jestor e ir para o Pipeline de Cadastro de Acesso
O que fazer
Entrar no Jestor e abrir o pipeline de cadastro
Como fazer
Acesse app.jestor.com → entre com suas credenciais → localize OPS | Pipeline de Cadastro de Acesso
ℹ️ A tabela, o pipeline e os formulários já estão configurados. Não crie novos campos ou estruturas sem alinhamento com a equipe.
2
Localizar o card no pipeline
O que fazer
O cliente preenche o formulário e o card já chega criado automaticamente no pipeline com status Nova Solicitação. A colaboradora não preenche o formulário — apenas localiza e trata o card.
Como fazer
Acesse o pipeline → localize o card do cliente em Nova Solicitação → abra o card para verificar os dados
SE
dados completos e corretos
→
ENTÃO
O responsável da unidade move o card para Cadastro Autorizado
SE
dados incompletos ou incorretos
→
ENTÃO
Não avançar — acionar o responsável da unidade antes de qualquer ação
⚠️ Só inicie o processo no Control ID após o card estar em Cadastro Autorizado.
3
Selecionar o formulário correto
O que fazer
Usar o formulário correspondente ao tipo de cliente para novos cadastros
Formulários disponíveis
🏢 Formulário — Empresa XP abrir → 🏢 Formulário — Unidades de Coworking abrir → 🏢 Formulário — Prudential abrir →
⚠️ Se o cliente não se encaixa em nenhum dos 3 formulários, não avance — acione a equipe antes de cadastrar.
4
Acompanhar o pipeline
O que fazer
Acompanhar e mover o card conforme o andamento do processo
Como fazer
Acesse o pipeline → localize o card do cliente → mova para a etapa correta
🗂️ Pipeline — Cadastro de Acesso abrir →
Etapas do pipeline
🔵 Nova Solicitação
Automação. Card criado ao enviar o formulário. Não mover manualmente.
✅ Cadastro Autorizado
Movido manualmente pelo responsável da unidade após validar os dados do formulário. Indica que o cadastro está pronto para o Control ID.
⏳ Control ID Pendente
Movido manualmente ao iniciar o cadastro no Control ID. Processo em andamento.
📷 Foto não aprovada
Automação: ao mover o card para esta etapa, um e-mail é disparado ao cliente solicitando nova foto.

Quem aprova a foto: o próprio Control ID — não é decisão da colaboradora.

Quando o cliente enviar a nova foto:
  1. A foto chega por e-mail
  2. Acesse a foto pelo card do cadastro no pipeline
  3. Submeta a nova foto no Control ID
  4. Aguarde a validação do sistema
✅ Aprovada → mover para Control ID Concluído
❌ Rejeitada → manter em Foto não aprovada — automação reenvia e-mail
✅ Control ID Concluído
Movido manualmente após cadastro e foto aprovados no Control ID. Se não há etapa de condomínio, avançar direto para Finalizado.
🏛️ Condomínio Pendente
O cadastro no condomínio sempre acontece, mas o canal varia. Consulte a tabela de perfil operacional para saber como proceder:
📧 E-mail → automação dispara ao mover o card. Nenhuma ação manual necessária.
📱 App ou Portal → não é responsabilidade do time de atendimento.
Após concluir → mover o card para Condomínio Concluído.
✅ Condomínio Concluído
Atualizado automaticamente após o envio do e-mail ao condomínio. Verificar e avançar para Finalizado.
🏁 Finalizado
Processo completo. Acesso liberado e tudo registrado.
🚫 Negado
A decisão é do responsável da unidade, não da colaboradora. As regras de liberação devem ser alinhadas previamente com a liderança. Em caso de dúvida, consulte o responsável antes de mover o card.
👤 Pessoa Desativada
Acionado quando a empresa do cliente informa desligamento.
  1. Mover o card para Pessoa Desativada
  2. Consultar o e-mail do condomínio na tabela de perfil operacional → coluna E-mail
  3. Enviar e-mail manualmente ao condomínio informando o bloqueio
  4. Acessar o Control ID → Cadastro e Pessoas
  5. Usar a janela de busca → digitar o nome do cliente
  6. Clicar na lixeira vermelha ao lado do registro
  7. Clicar no botão Inativar para confirmar
⚠️ Não inative sem confirmação da empresa. A solicitação de desligamento precisa ter chegado antes de qualquer ação.
🔁 Duplicado
Cliente já existe no Control ID. Não crie nada — encerrar o processo aqui.
ℹ️ Etapas com automação (Nova Solicitação, Foto não aprovada, Condomínio Pendente → Concluído) não precisam de ação manual além de mover o card para o status correto que dispara a automação.

Como fazer
  1. Clicar na aba Grupos e Empresas

⚠️ As empresas já estão cadastradas — não crie novas. Se não encontrar, confira o nome no Jestor e tente a busca com um termo mais curto.
6
Salvar e atualizar o pipeline
Como fazer
Salvar o cadastro no Control ID → voltar ao Jestor → mover o card para Control ID Concluído
ℹ️ Só mova o card para Control ID Concluído após salvar tudo e confirmar que a foto foi aprovada.

⚠️Erros Comuns
▼
🔴 Cadastro duplicado no Jestor
Por que acontece
Não foi feita a busca pelo CPF antes de criar novo registro
Como evitar
Sempre busque o CPF/CNPJ antes de criar. Sem exceção.
Como corrigir
Identificar qual registro está correto → excluir ou mesclar o duplicado → atualizar no Control ID se necessário
🔴 Acesso negado mesmo após cadastro
Por que acontece
Na maioria dos casos: foto reprovada ou não atualizada no Control ID. Em alguns casos: erro de sincronização entre o Control ID e a controladora física presente no local — o cadastro existe no sistema, mas ainda não foi transmitido para o equipamento.
Como evitar
Sempre validar a foto no Control ID antes de mover o card para Control ID Concluído.
Como corrigir
Primeiro, verificar a foto no Control ID → se reprovada, mover o card para Foto não aprovada. Se a foto estiver aprovada, aguardar a sincronização com a controladora física ou acionar o responsável técnico.
🔴 Empresa errada em Grupos e Empresas
Por que acontece
Empresa errada selecionada em Grupos e Empresas no Control ID
Como evitar
Confirmar a empresa com os dados do Jestor antes de salvar
Como corrigir
Editar o cadastro no Control ID → aba Grupos e Empresas → desvincular a empresa errada → vincular a correta → salvar
🔄Situações Especiais
▼
🔴 Cliente encontrado no Control ID (duplicado)
  1. Não crie nada no Control ID
  2. Volte ao Jestor imediatamente
  3. Localize o card do cliente no pipeline
  4. Mova o card para o status Duplicado
  5. Encerre o processo — não avance para nenhuma outra etapa
⚠️ Não tente corrigir ou mesclar registros no Control ID por conta própria. Sinalize como duplicado e aguarde orientação se necessário.
✏️ Alteração de dados de cliente já cadastrado
  1. Altere primeiro no Jestor
  2. Salve e confirme os novos dados
  3. Replique a alteração no Control ID
  4. Verifique se o acesso continua funcionando após a alteração
⚠️ Nunca altere só em um sistema. Sempre atualize os dois.
🚨 Urgência (acesso precisa funcionar agora)
  1. Cadastre primeiro no Control ID com os dados mínimos disponíveis
  2. Libere o acesso temporariamente
  3. Imediatamente após: complete o cadastro no Jestor
  4. Atualize o Control ID com os dados completos
  5. Sinalize para o time que houve um cadastro emergencial
⚠️ Acesso emergencial não dispensa o cadastro completo posterior.
💻 Falha no sistema (Jestor ou Control ID fora do ar)
  • Registre os dados do cliente em local seguro (planilha de contingência ou papel)
  • Não tente cadastrar sem o sistema — isso evita registros incompletos
  • Assim que o sistema voltar, realize o cadastro completo
  • Comunique a equipe sobre o atraso
💡Boas Práticas
▼
  • 🔍Sempre busque o CPF/CNPJ no Jestor antes de criar qualquer cadastro novo
  • 📋Use o checklist desta SOP antes de iniciar — mesmo quando estiver com pressa
  • 💬Em caso de dúvida, pergunte antes de salvar — é mais rápido do que corrigir depois
  • 🧪Sempre teste o acesso após liberar — não assuma que funcionou
  • 📝Registre exceções e situações incomuns para atualizar esta SOP
  • 🚫Não compartilhe acesso de cadastro com outra pessoa — cada operadora usa sua credencial
  • 🔄Se alterar algum dado em um sistema, atualize imediatamente no outro
  • ⚡Em urgências: cadastre o mínimo para funcionar, mas complete depois sem falta
