📋Pré-Requisitos
▼
🔴 Obrigatório
  • CPF ou CNPJ do cliente (válido)
  • Nome completo do cliente
  • Unidade / espaço associado ao acesso
  • Contrato ativo ou confirmação de vínculo
  • Data de início do acesso
  • Foto do cliente — enviada com o formulário (Control ID) ou não necessária (UniFi)
  • E-mail do cliente (obrigatório no UniFi)
🟡 Importante
  • Telefone de contato
  • Data de fim do acesso (se aplicável)
⚠️ Todos os dados chegam preenchidos pelo cliente via formulário. O CPF é obrigatório para o cadastro no condomínio.
🟦Passo a Passo — Jestor
▼

Sistema de gestão interna. Ponto de entrada obrigatório para qualquer cadastro.

1
Acessar o Jestor e ir para o Pipeline de Cadastro de Acesso
Como fazer
Acesse app.jestor.com → entre com suas credenciais → localize OPS | Pipeline de Cadastro de Acesso
🗂️ Pipeline — Cadastro de Acesso abrir →
2
Localizar o card no pipeline
O que fazer
O cliente preenche o formulário e o card já chega criado automaticamente com status Nova Solicitação. A colaboradora não preenche o formulário — apenas localiza e trata o card.
SE
dados completos e corretos
→
ENTÃO
Responsável da unidade move para Cadastro Autorizado
SE
dados incompletos ou incorretos
→
ENTÃO
Não avançar — acionar o responsável antes de qualquer ação
⚠️ Só avance para o sistema de acesso após o card estar em Cadastro Autorizado.
3
Identificar o sistema da unidade
O que fazer
Consultar a tabela de perfil operacional para saber qual sistema a unidade usa e se há recepcionista
📋 Tabela — Perfil Operacional abrir →
Com base no perfil operacional, siga para:
SE
sistema = UniFi
→
ENTÃO
Acionar Ops → seção UniFi
SE
sistema = Control ID + tem recepcionista
→
ENTÃO
Atendimento segue → seção Control ID
SE
sistema = Control ID + sem recepcionista
→
ENTÃO
Acionar Ops → seção Control ID
4
Selecionar o formulário correto
🏢 Formulário — Empresa XP abrir → 🏢 Formulário — Unidades de Coworking abrir → 🏢 Formulário — Prudential abrir →
⚠️ Se o cliente não se encaixa em nenhum dos 3 formulários, não avance — acione a equipe antes de cadastrar.
5
Acompanhar o pipeline
Etapas do pipeline
🔵 Nova Solicitação
Automação. Card criado ao enviar o formulário. Não mover manualmente.
✅ Cadastro Autorizado
Movido pelo responsável da unidade após validar os dados. Pronto para o sistema de acesso.
⏳ Control ID / UniFi Pendente
Movido ao iniciar o cadastro no sistema de acesso. Processo em andamento.
📷 Foto não aprovada
Automação: e-mail disparado ao cliente. Quem aprova: o próprio Control ID. Aplicável somente a unidades Control ID.
✅ Aprovada → Control ID / UniFi Concluído
❌ Rejeitada → manter aqui — automação reenvia e-mail
✅ Control ID / UniFi Concluído
Movido após cadastro concluído no sistema de acesso. Se não há etapa de condomínio, avançar para Finalizado.
🏛️ Condomínio Pendente
Canal varia por unidade. Consulte a tabela de perfil operacional.
📧 E-mail → automação dispara ao mover o card.
📱 App ou Portal → não é responsabilidade do time de atendimento.
✅ Condomínio Concluído
Atualizado automaticamente após envio do e-mail. Avançar para Finalizado.
🏁 Finalizado
Processo completo. Acesso liberado e tudo registrado.
🚫 Negado
Decisão do responsável da unidade. Regras de liberação devem ser alinhadas com a liderança previamente.
👤 Pessoa Inativa
Acionado quando a empresa informa desligamento.
  1. Mover card para Pessoa Inativa
  2. Consultar e-mail do condomínio na tabela de perfil operacional → coluna E-mail
  3. Enviar e-mail ao condomínio informando o bloqueio
  4. Inativar no sistema de acesso da unidade
⚠️ Não inative sem confirmação da empresa.
🔁 Duplicado
Cliente já existe no sistema de acesso. Não criar nada — encerrar o processo aqui.
ℹ️ Etapas com automação não precisam de ação manual além de mover o card para o status correto.
🟣Passo a Passo — UniFi
▼

Sistema de controle de acesso para unidades UniFi. Executado exclusivamente por Ops (Ludmilla ou Luana).

🟣 O UniFi avisa pelo e-mail se a pessoa já está cadastrada. O e-mail é obrigatório.
1
Acessar o UniFi
O que fazer
Entrar no portal UniFi com suas credenciais
🔐 Login — UniFi abrir →
⚠️ Acesso restrito a Ops. Somente Ludmilla e Luana têm credenciais.
2
Selecionar a unidade correta
Como fazer
Clique no seletor de site no menu superior esquerdo → selecione a unidade correspondente ao cadastro
⚠️ Confirme que está na unidade correta antes de criar o usuário.
3
Criar novo usuário
Como fazer
Painel lateral → Usuários ativos → clique no + azul → abre o formulário Criar novo usuário
Campos a preencher
Primeiro nomePrimeiro nome — conforme Jestor
SobrenomeRestante do nome — conforme Jestor
E-mailObrigatório. O sistema avisa se o e-mail já existe — não criar duplicata.
ℹ️ Se o UniFi alertar que o e-mail já existe, volte ao Jestor e mova o card para Duplicado.
4
Vincular ao grupo da unidade
Como fazer
Na seção Grupos do formulário → Selecionar grupos → selecionar o User Group padrão da unidade
Como saber o grupo correto
Consulte a tabela de perfil operacional — campo User Group padrão da unidade
⚠️ Sem o grupo vinculado, o cliente não terá as políticas de acesso da unidade aplicadas.
5
Criar e confirmar
Como fazer
Revisar os dados → clicar em Criar → confirmar que o usuário aparece na lista de Usuários ativos
6
Atualizar o pipeline no Jestor
Como fazer
Voltar ao Jestor → mover o card para Control ID / UniFi Concluído
ℹ️ Só mova o card após confirmar que o usuário está visível na lista do UniFi.

Executado por Atendimento (quando há recepcionista) ou por Ops (quando não há).

⚠️ Sempre busque o cliente antes de criar. Duplicata aqui gera conflito de acesso.
1
Acessar o Control ID
🔐 Login — Control ID abrir →
ℹ️ Use suas credenciais pessoais. Não compartilhe com outras colaboradoras.
2
Buscar se o cliente já existe
Como fazer
Menu lateral → Cadastros → Pessoas → campo Pesquisar → digitar nome e sobrenome
SE
pessoa encontrada
→
ENTÃO
Não crie nada. Jestor → card para Duplicado
SE
pessoa não encontrada
→
ENTÃO
Clicar em Nova Pessoa (botão azul, canto superior direito) → Passo 3
⚠️ Pesquise por nome e sobrenome — só o nome pode gerar falso negativo.
3
Preencher os dados da nova pessoa
NomeNome completo — idêntico ao Jestor
ObservaçõesFormato: [Unidade] - [Empresa] — Ex: Brooklin - Pipo
E-mail / Celular / MatrículaSe disponível no formulário
⚠️ Campo Observações é obrigatório. Siga o padrão de nomenclatura do sistema.
4
Adicionar e validar a foto
  1. Clicar no ícone de upload (nuvem azul)
  2. Selecionar a foto nos arquivos do computador
  3. Aguardar o upload → clicar no ícone de check (✔) para validar
SE
foto aprovada
→
ENTÃO
Continuar para o Passo 5
SE
foto reprovada
→
ENTÃO
Jestor → card para Foto não aprovada
5
Vincular grupo e empresa
  1. Aba Grupos e Empresas → botão Vincular
  2. Buscar no formato Be in [Unidade] - [Andar]° - [Empresa]
  3. Clicar na seta azul → mover para Selecionados → Salvar
⚠️ Empresas já estão cadastradas — não crie novas.
6
Salvar e atualizar o pipeline
Como fazer
Salvar no Control ID → Jestor → card para Control ID / UniFi Concluído

⚠️Erros Comuns
▼
🔴 Cadastro duplicado no Jestor
Como evitar
Sempre busque o CPF/CNPJ antes de criar. Sem exceção.
Como corrigir
Identificar o registro correto → excluir ou mesclar o duplicado → atualizar no sistema de acesso se necessário
🔴 Acesso negado mesmo após cadastro (Control ID)
Por que acontece
Foto reprovada ou erro de sincronização entre o Control ID e a controladora física.
Como corrigir
Verificar a foto → se reprovada, mover para Foto não aprovada. Se aprovada, aguardar sincronização ou acionar o responsável técnico.
🔴 Empresa errada em Grupos e Empresas (Control ID)
Como corrigir
Editar no Control ID → Grupos e Empresas → desvincular a errada → vincular a correta → salvar
🟣 E-mail duplicado no UniFi
Por que acontece
O cliente já possui cadastro ativo no UniFi daquela unidade
Como corrigir
Não criar — voltar ao Jestor e mover o card para Duplicado
🔄Situações Especiais
▼
🔴 Cliente encontrado no sistema de acesso (duplicado)
  1. Não crie nada no sistema de acesso
  2. Jestor → card para Duplicado
  3. Encerre o processo
⚠️ Não tente corrigir registros por conta própria. Sinalize como duplicado.
✏️ Alteração de dados de cliente já cadastrado
  1. Altere primeiro no Jestor → salve e confirme
  2. Replique a alteração no sistema de acesso da unidade
  3. Verifique se o acesso continua funcionando
⚠️ Nunca altere só em um sistema. Sempre atualize os dois.
🚨 Urgência (acesso precisa funcionar agora)
  1. Cadastre no sistema de acesso com os dados mínimos disponíveis
  2. Libere o acesso temporariamente
  3. Complete o cadastro no Jestor imediatamente após
  4. Sinalize para o time
⚠️ Acesso emergencial não dispensa o cadastro completo posterior.
💻 Falha no sistema
  • Registre os dados em local seguro
  • Não tente cadastrar sem o sistema — isso evita registros incompletos
  • Assim que voltar, realize o cadastro completo e comunique a equipe
💡Boas Práticas
▼
  • 📋Consulte sempre a tabela de perfil operacional antes de ir ao sistema de acesso
  • 🔍Sempre busque o CPF/CNPJ no Jestor antes de criar qualquer cadastro novo
  • ✅Use o checklist desta SOP antes de iniciar — mesmo quando estiver com pressa
  • 💬Em caso de dúvida, pergunte antes de salvar — é mais rápido do que corrigir depois
  • 🧪Sempre teste o acesso após liberar — não assuma que funcionou
  • 📝Registre exceções e situações incomuns para atualizar esta SOP
  • 🚫Não compartilhe credenciais — cada pessoa usa o seu próprio login
  • 🔄Se alterar algum dado em um sistema, atualize imediatamente no outro
  • ⚡Em urgências: cadastre o mínimo para funcionar, mas complete depois sem falta
